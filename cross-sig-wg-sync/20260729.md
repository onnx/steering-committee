## 2. PyTorch Foundation Membership Discussion

### Background
- ONNX is currently an **ecosystem member** of PyTorch Foundation (listed on landscape webpage only)
- Previously joined LFAIData Foundation (Linux Foundation project)
- PyTorch Foundation recently inquired about ONNX becoming a **hosted project** (higher tier)

### Benefits of Hosted Project Status
- Additional funding opportunities
- Marketing support
- Keynote opportunities at PyTorch conferences
- Increased visibility and community access

### Requirements Met
- Achieved OpenSSL Silver Batch status 2-3 weeks ago
- Now eligible to apply for hosted level membership

### Potential Concerns
- **Neutrality Questions:** Association with PyTorch could signal bias (though ONNX is framework-agnostic, working with TensorFlow, PyTorch, etc.)
- **Foundation Relationship:** Would need to leave LFAIData Foundation
- **Unclear Funding Details:** Financial benefits from PyTorch Foundation not explicitly confirmed

### Discussion Summary
- *Person 1* Skeptical unless advantages clearly outweigh disadvantages; no compelling reason to switch without strong benefits
- *Person 2* Comfortable with ecosystem status but less convinced about hosted level; noted financial benefits exist but not the primary motivation
- *Person 3* Supportive if resources/support increase; questioned effort required for transition
- *Person 4* Identified chicken-and-egg problem—need resources to leverage foundation benefits, but won't get resources until joining
- *Person 5* Offered to introduce Eclipse Foundation as alternative; noted neutrality concerns are valid

### Funding Notes
- Community meetups currently sponsored by companies (Intel, AMD, Microsoft, NVIDIA)
- Christian Bourjau reported Quanco (company) willing to co-sponsor European edition
- Recommendation: Lobby employers for funding rather than relying on foundation

### Decision Process
- **Steering Committee Vote:** Expected in ~2 weeks
- **Community Input:** This meeting served as feedback gathering for final steering committee decision
- Formal application process required if decision is to proceed

## 3. Community Survey Results

### Survey Overview
- **Duration:** February–April 2026 (still open with occasional responses)
- **Responses:** 33 respondents out of a larger community
- **Questions:** 12 total
- **Presenter:** Christian Bourjau

### Key Findings

#### User Demographics
Most respondents use ONNX for:
1. **Inference/Deployment** (top usage)
2. **Model Optimization/Graph Optimization**
3. **Training** (some overlap with optimization terminology)
4. **Tooling and Model Modification**
5. **Research and Experimentation**

#### Main Pain Points (3 Categories)

**1. Specification Evolution**
- Need for better LLM and related operator support
- Semantic clarity issues in specification
- Missing operators for emerging fields (e.g., statistical operators)
- Quality and correctness improvements needed

**2. Runtime Support**
- Implementations need better conformance to spec
- Performance optimization important
- End-to-end perspective needed (generation alone insufficient)

**3. Converters and Tooling**
- Tooling is scattered across frameworks
- Documentation unclear
- Buggy implementations
- Need better scikit-learn integration specifically
- Users want to modify existing model graphs

### Solutions Proposed

**RFC (Request for Comments) Process**
- Recently introduced to streamline specification discussions
- Crystallization point for larger ecosystem development
- Goal: Improve responsiveness on feature decisions

**Test Coverage and Quality**
- Improve operator test cases and coverage
- Critical for safe AI-assisted code generation
- Conformance Test Suite needed by end of 2026
- Will help uncover missing kernels and bugs
- Christian's parametric test cases project moving into ONNX namespace

**AI-Assisted Development**
- Use AI coding agents to address technical debt
- Improve operator documentation
- Reduce tedium in codebase improvements
- **Caution:** Need rigorous verification; can't blindly accept AI-generated changes

**Skills Files and Documentation**
- Develop skill files for operators documenting constraints and design patterns
- Simplify homepage documentation (currently verbose)
- Improve converter documentation and user experience

### Overall Sentiment
- **Neutral to Positive:** Most respondents satisfied with ONNX
- **Pain Points:** Friction exists for newcomers and complex workflows
- **Rough Cases:** Users report frustration with specific conversion scenarios but overall ecosystem is viewed favorably

## Meeting Outcome

The meeting served primarily as feedback gathering for a pending steering committee decision on PyTorch Foundation membership. While no final decision was made during the meeting, consensus emerged that:
1. Current ecosystem status is acceptable but limited
2. Hosted status benefits are uncertain and require clarification
3. Community should lobby employers for meetup funding rather than relying on foundations
4. Survey results validate community needs around specification evolution, tooling, and runtime support
